## ASO Notes

- **Keywords são global pool**: usa só palavras que NÃO estão no title nem subtitle. Apple cruza title + subtitle + keywords pra search.
- **Subtitle aparece em search results** — atua quase como hook secundário, importante punchy.
- **Promotional Text** edita SEM precisar nova review — útil pra campanhas sazonais.
- **What's New** mostra full text expandido na update sheet — header com data + versão.

## Privacy Labels (App Privacy section)

Use esse mapping no formulário App Privacy (todos sem tracking):

| Categoria | Coletado? | Linked to user? | Tracking? |
|-----------|-----------|-----------------|-----------|
| Contact Info → Email | ✅ Yes | Linked | ❌ No |
| User Content → Photos and Videos | ✅ Yes | Linked | ❌ No |
| User Content → Audio (video sound) | ✅ Yes | Linked | ❌ No |
| User Content → Other (workout posts, captions) | ✅ Yes | Linked | ❌ No |
| Identifiers → User ID (Supabase auth) | ✅ Yes | Linked | ❌ No |
| Identifiers → Device ID (push token) | ✅ Yes | Linked | ❌ No |
| Location → Coarse (gym selection — optional) | ✅ Optional | Linked | ❌ No |
| Usage Data → App interactions (analytics interno) | ✅ Yes | **Not Linked** | ❌ No |
| Diagnostics → Crash data | ✅ Yes | Not Linked | ❌ No |

**Important**: marcar como "Data Not Used for Tracking" explicitamente.

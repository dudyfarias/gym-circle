### Phase B — Stories contínuos 🔄 PARCIAL

#### B1 — Stories cross-author Instagram-like ✅ LIVE (commit `09ef246`)

**Divergência do plano original:** o plano assumia criar `useStoryQueue` hook + tipo `EnrichedStoryGroup`. O código real já tem `StoryGroup{id,author,stories}` + `social.storyGroups` + `social.actions.openStory(storyId)` que auto-resolve grupo. Solução **in-place** no `GymCirclePreview` — sem hook novo.

**Antes:** ao terminar último story do autor, `closeStory()` fechava o viewer. User precisava reabrir manualmente o próximo.

**Agora:**
- `openNextStory`: próximo no autor → senão **primeiro do próximo autor** → senão `closeStory()` (fim absoluto).
- `openPreviousStory`: anterior no autor → senão **último do autor anterior**.
- `hasNext`/`hasPrevious` consideram cross-author (`hasNextStoryOrAuthor`/`hasPrevStoryOrAuthor`).

**O viewer só fecha no fim absoluto da fila.** Comportamento Instagram/Threads.

#### B2 — Remove `key={story.id}` do StoryViewer + pre-decode swap ⏳ PENDENTE

**Por que importa:** o `key={story.id}` causa unmount+remount completo do `StoryViewerContent` a cada troca de story. Entre os 2 frames, o `bg-black/94` do parent aparece — é exatamente o flash preto que o spec quer eliminar.

**Por que é arriscado:** sem o `key=`, todos os states internos (replyDraft, menuOpen, shareOpen, heartBurst, mediaLoaded) precisam de reset manual via `useEffect([story.id])`. Esquecer um state = bug visual sutil.

#### B4 — Smoke iPhone manual ⏳ PENDENTE

Validar stories contínuos no app real após B2.

- `inputFocused` → `false`
- `mediaLoaded` → re-check `hasImageLoaded(story.imageUrl)`

Plus o pre-decode pattern (similar ao `PinchZoomImage` da Sprint 3.6):
```ts
useEffect(() => {
  if (hasImageLoaded(story.imageUrl)) {
    setMediaLoaded(true);
    return;
  }
  setMediaLoaded(false);
  let cancelled = false;
  const img = new window.Image();
  img.decoding = "async";
  img.src = story.imageUrl;
  img.decode().then(() => {
    if (cancelled) return;
    markImageLoaded(story.imageUrl);
    setMediaLoaded(true);
  }).catch(() => {});
  return () => { cancelled = true; };
}, [story.imageUrl]);
```
